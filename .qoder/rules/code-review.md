# code-review

## 复盘原则

1. 检查是否存在不合理的方案设计；
2. 检查是否存在过度设计；
3. 检查是否存在遗留代码；
4. 检查是否存在显著逻辑错误、或严重缺陷；
5. 借助 KISS 原则检查文案合理性；
6. 借助 DRY 原则检查代码重复性；
7. 借助 SOLID 原则检查代码可维护性；
8. 借助 YAGNI 原则检查代码必要性；
9. 借助 CLEAN CODE 原则检查代码可读性；
10. 借助 COHERENT 原则检查代码一致性；
11. 借助 CONCISE 原则检查代码简洁性；

## 第一性原则

始终保持第一性原则分析。
